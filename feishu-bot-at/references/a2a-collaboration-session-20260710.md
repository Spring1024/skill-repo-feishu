## 关键发现

### 1. Open ID 混淆问题
- **现象**：波比的 open_id 有两个不同值
  - `ou_cb099e84ffa7033c4a51d4b332f4340f` — 来自 @ 标记（open_id 格式）
  - `ou_a6ee2e72ade4a7d629f6873fbd55e76f` — 来自 API 返回（open_id 格式）
- **结论**：两者都可能有效，取决于飞书 API 的调用方式。建议始终通过 API 获取最新值。

### 2. 芙蓉的 on_ vs ou_ 问题
- **现象**：芙蓉的会话密钥中包含 `on_31f5a8cdc9c5aebdee987d9297dcbc74`（错误的前缀）
- **正确值**：`ou_f18431cc99607e49e29094cb589bb5c6`（通过 API 获取）
- **教训**：飞书 open_id 格式永远是 `ou_` 开头，不是 `on_`。

### 3. 群聊配置影响
- `FEISHU_REQUIRE_MENTION=false` — 允许 bot 响应所有消息（不限于 @mention）
- `FEISHU_ALLOW_BOTS=mentions` — 允许 bot 之间互相 @
- 组合效果：bot 可以主动发送 @ 消息给其他 bot，且不会被 @ 限制

### 4. 消息格式规范
- 每条消息必须以类型标签开头（===✅️回复✅️===, ===📋任务📋=== 等）
- 身份标识格式：`[Bot:芙蓉]`
- @ 其他 bot 时必须使用 `feishu-bot-at` 技能的脚本，不能直接用文本 `<at>` 标签
